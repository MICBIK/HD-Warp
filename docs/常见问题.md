# ❓ HD Warp 常见问题 (FAQ)

---

## 🔧 安装相关

### Q: macOS 提示"应用已损坏，无法打开"？

**A:** 这是 macOS 的 Gatekeeper 安全机制导致的，执行以下命令解决：

```bash
sudo xattr -rd com.apple.quarantine /Applications/HD\ Warp.app
```

然后重新打开应用即可。

---

### Q: 如何检查 mitmproxy 是否安装成功？

**A:** 在终端执行：

```bash
mitmdump --version
```

如果显示版本号（如 `Mitmproxy: 10.x.x`），则安装成功。

---

## 🔐 证书相关

### Q: 启动网关后 Warp 无法使用 AI 功能？

**A:** 90% 的情况是证书未正确安装或信任。请检查：

1. 打开「钥匙串访问」应用
2. 搜索 `mitmproxy`
3. 确认证书存在且显示**蓝色 ✓**
4. 如果不是蓝色，双击证书 → 信任 → 选择「始终信任」

---

### Q: 证书显示"此证书不受信任"？

**A:** 需要手动设置信任级别：

1. 双击证书
2. 展开「信任」部分
3. 将「使用此证书时」改为「始终信任」
4. 关闭窗口，输入密码确认

---

## 🌐 网关相关

### Q: 遇到 403 Forbidden 错误？

**A:** 403 通常表示账号临时受限，这是正常现象：

- 继续发送请求，系统会自动切换账号
- 等待几秒后重试
- 多次尝试后即可恢复

---

### Q: 显示 502 Bad Gateway？

**A:** 502 通常是网络问题：

1. 检查网络连接
2. 尝试重启网关
3. 如果使用代理，检查上游代理是否正常
4. 系统会自动重试 502 错误

---

### Q: 初次启动时 AI 请求失败？

**A:** 首次启动需要验证所有账号状态：

1. 观察左下角的「就绪账号」数量
2. 等待数量稳定（不再变化）
3. 再开始使用 Warp

验证过程中可能出现临时错误，这是正常现象。

---

### Q: 网关启动后端口被占用？

**A:** 修改默认端口：

1. 进入「网关中心」
2. 在「端口配置」中修改监听端口
3. 重新启动网关

默认端口是 9528，可以改为其他未被占用的端口。

---

## 👤 账号相关

### Q: 额度显示不正确？

**A:** 额度信息会在使用过程中自动同步，也可以手动刷新：

1. 进入「账号池」页面
2. 点击「刷新额度」按钮
3. 等待刷新完成

---

### Q: 如何获取 API Key？

**A:** 在 Warp IDE 中获取：

1. 打开 Warp
2. 进入 Settings → Account
3. 点击「Create API Key」
4. 复制生成的 Key（格式：`wk-xxx...`）

---

### Q: 账号显示"封禁"状态？

**A:** 账号可能被官方封禁：

- 系统会自动跳过封禁账号
- 封禁账号不会影响其他账号使用
- 可以在「账号池」中删除封禁账号
